Image classification

See ImageClassification’s for complete documentation of its arguments

Let’s explore what you can do with the Method interface implemented. We’re using DLDatasets.jl to access ImageNette, a small image classification dataset. Install the package using

]add https://github.com/lorenzoh/DLDatasets.jl

ENV["DATADEPS_ALWAYS_ACCEPT"] = "true"
ENV["CI"] = "false"

using DLPipelines
using DLDatasets
using LearnBase: getobs

With the packages imported, loading the dataset is straightforward:

dataset = DLDatasets.loaddataset(ImageNette, "v2_160px")

FileDataset with 13394 files

As you can see, every observation consists of an image and a category label:

image, category = getobs(dataset, 4000)

Image classification Method

With the dataset ready, we can create an instance of ImageClassification.

Note that you could use any image classification dataset, as long as getobs(ds, idx) returns an image and a category.

method = ImageClassification(categories, sz = (128, 128))

ImageClassification() with 10 categories.

We can now use this method with a Context to transform the data. The image is encoded as a normalized 3D-array:

x = encodeinput(method, Training(), image)
summary(x)

LoadError("string", 1, UndefVarError(:image))

And the category is one-hot encoded:

y = encodetarget(method, Training(), category)

LoadError("string", 1, UndefVarError(:category))

You can also use MethodDataset to create a wrapper around your existing dataset that directly returns encoded observations:

methoddataset = MethodDataset(dataset, method, Training())
x, y = getobs(methoddataset, 1)
summary.((x, y))
